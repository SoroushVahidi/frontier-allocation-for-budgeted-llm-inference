# Results and artifacts guide

This guide summarizes which artifacts are manuscript-facing, diagnostic, or provenance-only. It is anonymous and claim-boundary focused.

## Artifact classes

- **Canonical paper-facing:** regenerated by `scripts/paper/run_all_neurips_paper_artifacts.py`; use for headline manuscript claims.
- **Appendix/supporting:** useful for robustness, audits, and bounded context; cite with caveats.
- **Diagnostic-current:** current exploratory evidence for method design; not canonical paper evidence.
- **Provenance-only:** historical, partial, failed, Wulver, or environment-specific records; retain but exclude or scrub for anonymous supplements.

## Headline claim guardrails

- Learned scorer outputs are **diagnostic-current**, not canonical paper-facing.
- Non-math Wulver evidence is mixed/incomplete and should remain provenance/appendix-only unless rerun with domain checks.
- HGB scorer should not be recommended because it degraded on fresh validation.
- RF/pairwise learned scorer is the strongest current learned-scorer candidate.
- Do not use outputs containing API readiness, retry logs, cluster paths, or private task links in anonymous supplements without scrubbing.
