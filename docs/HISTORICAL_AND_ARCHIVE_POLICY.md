# Historical and archive policy

This repository contains material from more than one phase of the broader project. This note explains how to interpret that material safely.

## Historical material

Historical material includes:
- earlier manuscript-oriented notes from superseded directions,
- older binary revise-routing support materials,
- dated memos replaced by current canonical docs,
- provenance-only notes kept for traceability.

Historical material is still useful for:
- provenance,
- comparison with older directions,
- explaining why the current paper framing changed.

Historical material is **not** the default interpretation of the repository.

## Archive policy

If files or notes are no longer active but are worth preserving, they should move into an archive-style location or be clearly marked as historical/provenance-only.

Until a larger physical reorganization is done, use this rule:
- canonical = current truth,
- exploratory = active but not default,
- historical = preserved only for provenance.

## Practical rule for collaborators

When in doubt:
1. start from [`CANONICAL_START_HERE.md`](CANONICAL_START_HERE.md),
2. treat [`README.md`](README.md) and [`REPO_MAP.md`](REPO_MAP.md) as the current navigation layer,
3. use historical material only after you already understand the canonical project.
