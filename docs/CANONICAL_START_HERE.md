# Canonical start here

This is the fastest reliable entry point for collaborators, future manuscript work, and implementation agents.

## Current project identity

The repository’s canonical direction is:
- fixed-budget adaptive test-time compute allocation,
- cross-controller frontier allocation,
- next-step branch allocation over active branches,
- answer-group-level incumbent-versus-challenger commit control,
- and supervision-target quality for branch allocation.

This repository is **not** currently centered on the old binary revise-routing paper.

## Current repository stance

The current bounded repository stance is:

> **keep the broad diversity/aggregation family as the main line, treat dependence-aware incumbent-versus-challenger commit control as the leading serious next method line, and use wrong-commit subtype analysis plus stronger exact-answer validation and real-model confirmation as the next reliability steps.**

The repository is currently in an **ICC-led method-hardening and evidence-strengthening phase**, not a broad nearby-experiment phase.
