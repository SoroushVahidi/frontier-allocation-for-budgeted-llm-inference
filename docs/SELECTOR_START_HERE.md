# Selector start here

This is the entry point for the current selector-first L1-defeat track.

## Scope

This track studies how to choose the final answer from candidate answer groups that have already been generated by the frontier/controller system.

It intentionally avoids mixing selector work with generation/coverage changes until selector-oracle diagnostics show whether the current candidate pool is sufficient.

## Current question

> Given the candidate answers already found by DR-v2 / OV / PRM variants, can a better selector choose the correct final answer often enough to close the gap to `external_l1_max`?

## Key distinction

Selector work answers:

> Did we find the correct answer but choose the wrong one?

Coverage work answers:

> Did we fail because the correct answer never appeared in the explored candidate pool?

Do not mix these two failure modes in the same experiment unless the experiment is explicitly designed to decompose them.

## Current required measurement

The next decisive measurement is a real trace-complete selector-oracle analysis with:

- gold answer,
- selected answer,
- correctness/exact match,
- candidate answer groups,
- normalized candidate answers,
- support/source metadata,
- optional OV/PRM scores,
- and method identity for paired comparison with `external_l1_max`.

The minimum headline diagnostic metrics are:

- `gold_present_rate`,
- `oracle_selector_accuracy`,
- `actual_selector_accuracy`,
- `selector_gap`,
- `L1_correct_ours_wrong_gold_present_count`,
- `L1_correct_ours_wrong_gold_absent_count`.

## Current decision rule

After a real selector-oracle analysis:

- If `gold_present_rate` and `oracle_selector_accuracy` are high, continue selector/reranker work.
- If `gold_present_rate` is low or oracle selector still cannot approach L1, shift to candidate generation / frontier coverage repair.
- If artifact schema prevents measurement, fix metadata emission before adding new selector ideas.

## What not to do

- Do not run large API experiments before the trace schema is confirmed.
- Do not claim OV/PRM selectors beat L1 without completed paired rows.
- Do not treat oracle selector as deployable; it is diagnostic only.
- Do not treat mock-backed verifier outputs as real verifier evidence.
- Do not add new selector variants before the real selector-oracle measurement works end-to-end.
